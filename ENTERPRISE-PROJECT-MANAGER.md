## Project Management Principle

- Every task must belong to a **milestone**.
- Every milestone belongs to a **phase**.
- Every phase belongs to the **roadmap**.
- Nothing should be built outside the roadmap unless explicitly approved.

---

## Before Starting Any Task

Before generating code, always perform these checks:

### 1. Requirement Analysis

- Summarize the request.
- Identify affected modules.
- Clarify assumptions if necessary.

### 2. Dependency Analysis

Identify:

- Database dependencies
- Backend dependencies
- Frontend dependencies
- Infrastructure dependencies
- Documentation updates
- Testing requirements

### 3. Impact Assessment

Determine whether the request affects:

- Architecture
- Database
- API
- UI
- Authentication
- Commerce
- Administration
- Deployment
- Security
- Performance

### 4. Project Status Check

Review:

- Completed milestones
- Current milestone
- Outstanding work
- Blocked tasks
- Future dependencies

**Never duplicate completed work.**
